Sample Workflow & Tools

Tools, using Adapt Discoverable Destinations, could work as follows:

  1. Retrieve list of discoverable destinations
  2. Search for relevant destination
    • Found? Navigate to destination
    • Not found? Notify user, stop
  3. Fetch content
  4. Process, summarize and present to user
Explain tool roles briefly.

Tool Integration using Model Context Protocol (MCP)

  • MCP Overview: An open protocol enabling AI agents to securely connect with external tools and data sources.
  • Roles:
    • MCP Clients: AI agents / LLM applications
    • MCP Servers: Tool providers (e.g., Semantic Web Tools)
  • Workflow: Agents discover destinations → plan navigation or retrieve content → process with LLMs, if required.
  • Note: MCP is one example, other protocols and frameworks may also be used.

MCP deployment models : How agents call these tools

  • Traditional MCP Server (remote):
    • Runs tools on a server; great for public pages
    • Limits: no access to user’s browser session and no live DOM/UI control.
  • WebMCP (in‑page):
    • The web page exposes tools to the agent via a JS API.
    • Strengths: access to full browser session and DOM control; ideal for authenticated, user‑specific flows.
