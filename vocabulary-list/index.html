Personalization Vocabulary List

Abstract

This document lists all vocabularies from personalization semantics. In this document, it divides the categories of vocabularies into three categories, they are Tokens, URI's and Text. Each vocabulary simply lists the description and values (Tokens vocabulary), and more information about each vocabulary can be found by "more info".

1. Vocabulary list

1.1 Tokens

  • action

    Description: The action attribute provides the context of a button. It is typically used on a button element or element with role="button".

    Values: add, brightness, buy, cancel, captions, channel, checkout, close, color, compose, confirm, connect, copy, cut, date, decrease, delete, download, drag, edit, escape, expand, forward, help, increase, label, left, loop, mediaPosition, mode, more, move, mute, new, next, open, order, paste, pause, play, previous, print, profile, properties, received, refresh, remove, reply, reset, resize, right, save, scroll, send, sent, settings, skip, submit, temperature, toc, undo, upload, voiceCommands, volume.

    See definition of action in module 'Personalization Semantics Content Module'.

  • destination

    Description: The destination attribute categorizes the target of a hyperlink.

    Values: about, accessibilityStatement, begin, chat, comment, contact, email, end, help, home, language, left, phone, post, products, related, right, services, signin, signout, sitemap, social, terms, tools.

    See definition of destination in module 'Personalization Semantics Content Module'.

  • field

    Description: The field attribute provides the context of a text input field such as a text box. It is typically used on an input of type text, or element with a corresponding role.

    Values: additionalName, additionalNameInitial, addressLine1, addressLine2, addressLine3, answer, area, bday, bdayDay, bdayMonth, bdayYear, ccCsc, ccExp, ccExpMonth, ccExpYear, ccName, ccNumber, ccType, city, comment, country, dateEnd, dateStart, email, familyName, fax, faxAreaCode, faxCountryCode, faxLocal, faxNational, gender, givenName, honorificPrefix, honorificSuffix, language, name, nickname, org, photo, position, postalcode, section-, sex, streetAddress, subject, tel, telAreaCode, telCountryCode, telExtension, telLocal, telNational, timeEnd, timeStart, topic, transactionCurrency, transactionAmount, url.

    See definition of field in module 'Personalization Semantics Content Module'.

  • simplification

    Description: The simplification attribute differentiates between critical features and less important features for simplification.

    Values: critical, medium (default), low.

    See definition of simplification in module 'Personalization Semantics Content Module'.

  • distraction

    Description: The distraction attribute should be used on detracting content, so that people who have problems keeping focus can turn the content off.

    Values: ad, animations, autoStarting, autoUpdating, chat, message, moving, overlay, popup, thirdParty.

    See definition of distraction in module 'Personalization Semantics Content Module'.

  • alternative

    Description: Would be used on an alternative content as an alternative to more detailed or difficult to understand content. alternative can be used on a span, div, link or image that servers as alternative content to it's direct parent element.

    Values: easyLangChunks, easyLangAll, lessOptions, literalImage, ImageSupplementary, realisticImage, simplifiedLang, easyLangSmallSentences, symbolic, vocab1000, vocab2000, vocab500, whiteSpace, interoprableSymbols, interoprableControls, userTested, understandableDesign, humanFocus, useableAuthentication, useableMenus, supportAPI, helpAndSupport, remindersAndSteps, noDataLoss. More values information can be found in Values for alternative content and content accessibility support.

    See definition of alternative in module 'Personalization Help and Support'.

  • helptype

    Description: Would be used on additional content to indicate the type of extra help provided. helptype can be used on a span, div, link or image that provides or redirects the user to additional help information.

    Values: automatedStep, automatedBot, dictionary, faqs, glossary, helpform, humanhelp, morehelp, moreinfo, simplified, thesaurus, tooltips, topicsearch, explain, extraHelp, feedback, triggerWarning, hideTimer, useableFeedback. More values information can be found in Values for types of help .

    See definition of helptype in module 'Personalization Help and Support'.

  • Status (from Step Indicator Usage)

    Description: The stepindicator defined values should be used whenever there is a sequence of tasks that the user is required to complete, so that users who have memory issues can keep track of the steps previously completed.

    See definition of status in module 'Personalization Tools'.

  • messageimportance

    Description: The messageimportance attribute is used to indicate the priority level of a message.

    Values: critical, medium (default), low.

    See definition of messageimportance in module 'Personalization Tools'.

1.2 URI’s

  • symbol

    Description: The symbol attribute identifies the concept for symbols.

    See definition of distraction in module 'Personalization Semantics Content Module'.

  • moreinfo

    Description: moreinfo provides users access to additional information about the current content in one of several formats.

    See definition of moreinfo in module 'Personalization Help and Support'.

  • extrahelp

    Description: extrahelp provides users access to additional information about the current action or process in one or more formats.

    See definition of extrahelp in module 'Personalization Help and Support'.

  • stepindicator – be indicated by an href

    Description: The stepindicator defined values should be used whenever there is a sequence of tasks that the user is required to complete, so that users who have memory issues can keep track of the steps previously completed.

    See definition of stepindicator in module 'Personalization Tools'.

1.3 Text

  • literal

    Description: literal provides alternative text for people who prefer content that does not use non literal text, such as metaphors.

    See definition of literal in module 'Personalization Help and Support'.

  • numberfree

    Description: numberfree provides alternative text for people who prefer content that does not use numerical concepts.

    See definition of numberfree in module 'Personalization Help and Support'.

  • easylang

    Description: easylang provides an alternative text of the content that is easier to read and understand.

    See definition of easylang in module 'Personalization Help and Support'.

  • explain

    Description: This attribute provides any information that helps the user anticipate the functionality, such as letting the user know what behavior will trigger an event.

    See definition of explain in module 'Personalization Help and Support'.

  • feedback

    Description: This attribute provides immediate feedback that can be shown or spoken when any event is triggered on a control.

    See definition of feedback in module 'Personalization Help and Support'.

  • messagefrom

    Description: The messagefrom attribute can be used to help identify who a message a user receives is from.

    See definition of messagefrom in module 'Personalization Tools'.

  • messagecontext

    Description: The messagecontext attribute can help clarify where a message is coming from.

    See definition of messagecontext in module 'Personalization Tools'.

  • messagetime (restricted)

    Description: The messagetime attribute helps clarify when the message was sent.

    See definition of messagetime in module 'Personalization Tools'.

  • Step (from Step Indicator Usage) numeric

    Description: The stepindicator defined values should be used whenever there is a sequence of tasks that the user is required to complete, so that users who have memory issues can keep track of the steps previously completed.

    See definition of stepindicator in module 'Personalization Tools'.
